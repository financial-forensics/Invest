     Вероятность: Высокая | Влияние: Критичное
     Митигация:
     • Change management программа с Q1 2026
     • Вовлечение key users в проектирование процессов
     • Обучение за 1 месяц до запуска каждого проекта
     • Система амбассадоров (champion users в каждом подразделении)
     • KPI руководителей включают adoption новых систем
     Владелец риска: CFO + HR директор
     
[R3] Перегрузка команды (2-3 проекта одновременно)
     Риск: Выгорание, снижение качества, срыв сроков
     Вероятность: Средняя | Влияние: Критичное
     Митигация:
     • Выделенная cross-functional команда на портфель (6-8 FTE)
     • Внешние консультанты на пиковые периоды
     • Строгая приоритезация: при конфликте ресурсов - Проект 1 важнее
     • Мониторинг загрузки команды еженедельно
     Владелец риска: Руководитель PMO
     
[R4] Недостижение ожидаемого экономического эффекта
     Риск: ROI ниже заложенного, проект формально "не окупился"
     Вероятность: Низкая | Влияние: Среднее
     Митигация:
     • Консервативные оценки эффекта (заложены -20% от реалистичного)
     • Tracking эффекта с первых месяцев после запуска
     • Корректирующие действия при отклонении >15%
     • Фокус на quick wins для демонстрации ценности
     Владелец риска: CFO
     
[R5] Технические проблемы интеграций между системами
     Риск: Системы не обмениваются данными корректно
     Вероятность: Средняя | Влияние: Среднее
     Митигация:
     • Proof of concept критичных интеграций до старта проекта
     • Использование стандартных API (не custom разработка)
     • Мониторинг интеграций 24/7 после запуска
     • Fallback план: ручная выгрузка/загрузка при сбое
     Владелец риска: Руководитель ДИТ
```

• Ежемесячно: Портфельные риски (на уровне Steering Committee)
• Квартально: Стратегические риски (на уровне Правления)

Критерии эскалации:
• Красный статус риска → немедленная эскалация CFO
• Задержка критической вехи >2 недель → эскалация Steering Committee
• Перерасход бюджета >10% → эскалация CFO и CEO
